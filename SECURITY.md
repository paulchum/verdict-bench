# Security Policy

## Supported Version

Security fixes are applied to the latest release and the default branch.

## Reporting

Please report vulnerabilities privately through GitHub's security-advisory interface for this repository. Do not include API keys, provider request payloads containing secrets, or non-public customer data in an issue.

## Benchmark Data

Public artifacts must contain synthetic Tau data or benchmark trajectories approved for release. CI runs secret scanning, but contributors remain responsible for reviewing generated logs before publication.
